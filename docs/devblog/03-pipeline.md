# 3. Pipeline Brief→Proof: флагман, собранный автономно

Самая амбициозная фича Verstak — **Pipeline**: превратить «бриф от заказчика» в готовый результат с доказательством, не разваливаясь на полпути. Поток:

```
Бриф → Plan → Execute → Verify → Proof Pack
```

Ты описываешь задачу человеческим языком. Агент строит план, выполняет шаги, прогоняет проверки (тесты/typecheck) и собирает **Proof Pack** — артефакт-доказательство того, что работа реально сделана и проверена, а не «модель сказала готово».

## Зачем это нужно

Обычный AI-агент заканчивает фразой «Готово!». Но «написал» ≠ «работает». Pipeline зашивает в продукт ту самую дисциплину, которой я следую в работе: цель формулируется через критерий приёмки **до** начала, а в конце предъявляется доказательство. Для агентств и B2B это hero-сценарий: клиенту уходит не «мы поправили», а Proof Pack с проверками.

## Как это собиралось — D1→D10 через `/loop`

Фишка в том, **как** Pipeline появился. Спека была разбита на 10 дней-шагов (D1–D10), и я прошёл их **автономно** в режиме `/loop` — без промежуточных «продолжать?». Каждый шаг — отдельный коммит, который откатывается:

- **D1** — миграция 22 `pipeline_runs` + CRUD в storage.
- **D2** — IPC `pipeline.ts` + мост в preload + типы в `api.d.ts`.
- **D3** — `PipelineWizard` (форма брифа) + чистая brief-логика (покрыта тестами).
- **D4** — стейт-машина в store + `PipelineBanner` + индексация шагов.
- **D5** — монтаж UI + оркестрация авто-send (Plan/Execute), race-safe.
- **D6** — Verify-панель в баннере на реальном `verifications.latest`.
- **D7** — Proof-шаг: генерация Proof Pack + preview + статус completed.
- **D8** — provider gate: Pipeline только для API-провайдеров (CLI one-shot не тянет multi-turn оркестрацию).
- **D9** — интеграционный progression-тест полного цикла.
- **D10** — First Win: кнопка «Попробовать Pipeline» в онбординге.

33 теста, миграция БД, IPC, store-стейт-машина, UI — и всё это зарелизено в **1.5.16**.

## Где была засада

Стейт-машина авто-send (D5) — самое скользкое место. Pipeline сам инициирует отправку сообщений (Plan → Execute → Verify) без участия человека, и тут легко словить гонку: два авто-send наложились, или send ушёл в уже закрытый чат. Спасло то, что незадолго до Pipeline мы вычистили класс race-багов через `SendRegistry` (Phase A тех-долга) — без этого Pipeline просто не был бы безопасным. Один рефакторинг расчистил дорогу другому.

Позже, уже 19.06, закрыли ещё 3 lifecycle-дыры: resume прерванного прогона, проброс `agentRunId`, авто-переход Execute→Verify. Флагман — это не «собрал и забыл», его лечишь по мере живой эксплуатации.

## Что осталось

- **Pipeline v1.1 Agency** — режим для агентств (через `WorkflowsPanel` + баннер). Нужна живая проверка.
- **Proof Pack → PDF + Telegram** — `printToPDF` (нужен Electron) + доставка. B2B/agency sales.
- **Remote Pulse** — TG-бот: статус прогона, approve write с телефона, доставка Proof Pack.

Pipeline — это про то, что AI-агент может быть не игрушкой, а инструментом с предъявляемым результатом.
