# Changelog

## 2.0.7 - 2026-07-15 (Надёжность провайдеров и моделей)

Релиз про то, чтобы Verstak никогда не «врал» о провайдерах и моделях: список моделей везде один и актуальный, устаревшая модель не уходит впустую, а интерфейс не обещает больше, чем движок реально умеет. Плюс две новые возможности для контроля над отправкой.

### Что нового для вас

- **Модель на один запрос.** Рядом с выбором модели — кнопка «🎯 модель на 1 запрос»: можно отправить одно сообщение через другого провайдера/модель, не меняя настройку чата. По умолчанию строгий режим — если выбранная модель недоступна, Verstak НЕ переключится молча на запасного, а честно об этом скажет.
- **Проверка моделей Grok Build.** В «Настройки → Модели» появилась кнопка «🩺 Проверить модели Grok Build». Если вы когда-то выбрали модель Grok, которой больше нет, — запрос больше не уходит «в никуда»: Verstak блокирует его и показывает доступные модели.
- **Единый список моделей.** Настройки и окно чата теперь берут список моделей из одного места. Раньше он был записан в нескольких местах и мог разъезжаться — из-за этого настройки два месяца предлагали модели Claude, которых уже нет. Больше такого не будет.
- **Честные подписи режимов.** Если провайдер работает через внешнюю программу (CLI), интерфейс это честно показывает и не обещает возможностей, которых у этого режима нет (откат правок, проверка выполнения и т.п.).

### Под капотом (невидимые «сторожа» надёжности)

- Список из 18+ провайдеров теперь описан в одном месте: добавить или потерять провайдера «наполовину» технически невозможно.
- Автоматические проверки не дают разъехаться: связке «интерфейс ↔ ядро», списку инструментов агента, переводам (русский/английский) и матрице возможностей каждого режима.
- Инструмент, который система «не знает», теперь отклоняется, а не выполняется вслепую.

Все изменения проверены автоматическими тестами и независимой придирчивой проверкой; несколько реальных ошибок были найдены и исправлены до релиза.

### Исправлено — критичное

- **Модели Claude в Настройках были мертвы.** В мае Anthropic перешёл на новые имена моделей, старые перестали работать. Исправление тогда затронуло движок, но **вкладку Настроек забыли** — и почти два месяца она предлагала три несуществующие модели Claude, а актуальную (`claude-sonnet-4-6`) выбрать было нельзя вовсе. Теперь список моделей Claude в Настройках совпадает с тем, что реально понимает движок.
- **Провайдер Codex OAuth не выбирался.** Выбираешь его в настройках — приложение молча подставляло Gemini (провайдер из 2.0.4 не был прописан во внутреннем списке). Теперь выбирается корректно.
- **Подписка Codex могла «выкинуть» из аккаунта.** Если при обновлении токена не удавалось сохранить его на диск (нет места, нет прав, файл держит антивирус) — доступ терялся и требовался повторный `codex login`. Теперь сессия продолжает работать, запись повторяется, а если сохранить так и не удалось — приложение честно предупреждает в ленте событий. Попутно: два параллельных чата больше не ломают токен друг другу, и токен не остаётся лежать во временном файле на диске.

### Исправлено — прочее

- Быстрое переключение между чатами могло применить настройку модели не к тому чату (гонка).
- Списки моделей Verstak Gateway и OpenRouter в Настройках были неполными — не хватало моделей, которые движок поддерживает.

### Под капотом

- Автоматический сторож качества кода: линтер на изменённые файлы в pre-commit, отчёты о здоровье кода и линте.
- Сторож против расхождения списков моделей/провайдеров между интерфейсом и движком — именно он поймал мёртвые модели Claude и невыбираемый Codex OAuth.
- Состязательные тесты: гонки при переключении чатов и отказы авторизации по подписке (22 сценария). Правки прошли 1–3 раунда независимого адверсариального ревью.

## 2.0.0 - 2026-07-11 (Security Hardening & Cleanup)

Аудит всей кодовой базы (discovery-воркфлоу, 70 подтверждённых находок) + **два раунда адверсариального ре-ревью каждого фикса** (нашли и закрыли 20+2 дефекта в самих фиксах, включая 2 регрессии). Фокус — доверие/безопасность (позиционирование Verstak) + чистка.

### Безопасность
- **Секреты не утекают в память агента.** Core-memory (постоянная память, инжектится в system prompt каждый ход) теперь редактирует секреты на записи, чтении, в возвращаемом результате и при вытеснении в архив — по всем путям, включая запись файла в обход.
- **Telegram не выгружает произвольные файлы.** `send_document` теперь отправляет только агент-сгенерированные артефакты (`.verstak/artifacts`), по реальному пути (защита от symlink). Раньше можно было выгрузить любой файл проекта (включая `.env`/`.git/config`).
- **Данные коннекторов помечены как недоверенные.** Вывод 1С/Telegram/GitHub/таблиц обрамляется непредсказуемым маркером — защита от инъекции команд через внешние данные (как у веб-доступа).
- **1С OData: защита от SSRF** через редиректы (как у HTTP-коннектора) — вредоносный endpoint не уведёт на внутренний/служебный адрес и не утащит авторизацию.
- **Telegram whitelist: fail-closed.** ⚠️ **Изменение поведения:** без настроенного `telegram_chat_whitelist` отправка теперь разрешена ТОЛЬКО в ваш `telegram_notify_chat_id` (раньше — в любой чат). Если пользуетесь отправкой в разные чаты — добавьте их в whitelist в настройках.
- **Скиллы с сервера** обязаны идти по https и не могут подменить встроенные/ваши локальные скиллы (защита от компрометации сервера).
- **Webview** встроенного браузера защищён (изоляция, без утечки привилегий).

### Приватность
- **«Удалить данные проекта» действительно удаляет всё.** Раньше оставались ~15 таблиц (история диалогов, «мозг проекта», сводки кода, планы) → это была утечка приватности + при повторном добавлении проекта воскрешал старый устаревший контекст. Теперь чистятся все таблицы проекта в одной транзакции.

### Исправлено
- Новый чат больше не наследует стоимость/ревью/превью прошлого чата.
- Плановый дозор и напоминания удаляются вместе с проектом (не крутятся вечно).
- Notification-напоминания не дублируются каждый тик.
- OAuth-токены коннекторов (Ozon Performance, SendPulse) привязаны к аккаунту — после смены креда не возвращается старый токен.

### Чистка
- Удалён мёртвый код: 11 неиспользуемых функций + 4 неиспользуемых компонента.

## 1.9.7 - 2026-07-10 (Long Sessions — вошло в релиз 1.9.9)

### Добавлено / Исправлено
- **CLI-крэш больше не тупик.** Прерванный CLI-прогон теперь предлагает повтор запроса одной кнопкой (раньше — только «показать что было»).
- **Авто-переключение подписочного аккаунта работает и для CLI.** Лимит активного аккаунта в CLI-прогоне переключает на следующий аккаунт пула, не теряя запрос (раньше срабатывало только на API-пути — то есть фактически не для того, ради чего делалось). С защитой от зацикливания при лимите без точного времени сброса.
- **Меньше износа БД на длинных сессиях.** Снапшот для восстановления не переписывается вхолостую и разрежается на длинных прогонах; порог размера подобран так, чтобы не отключать восстановление на моделях с большим контекстом.

### Добавлено
- **Проекция родного tool-таймлайна CLI.** Claude Code и Codex выполняют инструменты внутри бинаря (`Read`/`Edit`/`Bash` у Claude; `command_execution`/`file_change`/`mcp_tool_call`/`web_search` у Codex) — раньше эти события выбрасывались, в Timeline виднелся только финальный текст. Теперь проецируются как информационные tool-call (Verstak их НЕ переисполняет — CLI уже выполнил). Grok отложен: streaming-shape его tool-событий не подтверждён.
- **Честные runtime-ярлыки контроля.** Уровень контроля считается из provider+transport (не только transport): API = «Полный контроль», claude/codex CLI = «Наблюдаемый» (таймлайн виден, но исполнение вне Verstak), grok/gemini CLI = «Урезанный». Ни один CLI не выдаётся за full control. Бейджи в ModelPicker и Инспекторе запусков, i18n ru/en.
- **Control Envelope — git-якорь отката до CLI-прогона.** CLI пишет мимо undo-стека Verstak, поэтому перед каждым CLI-прогоном снимается честная точка отката через git (HEAD + недеструктивный `git stash create` снапшот грязных правок). - **permission-mode + guard секретов для claude-cli.** Режим агента зеркалится в `--permission-mode` (accept-edits→acceptEdits, plan→plan, bypass→bypassPermissions) — headless claude наконец умеет писать под контролем режима. Одновременно закрыты секреты в CLI: `--disallowedTools 'Read(**/.env)'`… для `.env`/`.ssh`/ключей/creds (зеркало isForbiddenPath, раньше guard жил только на API-пути).

### Известные ограничения
- Bash-эксфильтрация секретов из CLI (`cat .env`) флагами надёжно не режется (inherent-лимит) — сеть безопасности это Control Envelope (git-якорь).
